   - Session-based (lost on refresh)

   - No network latency
   - Users see changes immediately

   - No 404 errors possible (no external API calls)
   - No network failures (all in-memory)

   - Memory-limited only
   - No database queries during edits

   - Easy to debug (all state changes logged)
   - No complex async logic
